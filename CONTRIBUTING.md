# Contributing

## Adding a benchmark

1. Add your entry to [`benchmarks.yaml`](benchmarks.yaml) following the existing schema:

```yaml
- name: My Benchmark
  description: One or two sentence description of what it evaluates.
  category: coding # see existing categories in the readme
  links:
    # any key is valid: github, website, leaderboard, demo, etc.
    paper: https://arxiv.org/abs/...
    github: https://github.com/...
    website: https://...
```

2. Open a PR with just `benchmarks.yaml` - the README is auto-generated by CI, you don't need to touch it.

## Running locally

To generate the README without live stats:

```sh
uv run scripts/compile.py
```

To fetch live citation and star counts, create a `.env` file in the repo root:

```sh
S2_API_KEY=...       # optional, increases S2 rate limits
GITHUB_TOKEN=...     # required for star counts
```

Then run:

```sh
uv run scripts/compile.py --fetch
```

## Guidelines

- Do not push `README.md` directly - it is auto-generated by CI
- `description` should be 1–2 sentences, plain text.
- Use an arXiv link for the paper when available - this ensures automatic citation counts in the README.
